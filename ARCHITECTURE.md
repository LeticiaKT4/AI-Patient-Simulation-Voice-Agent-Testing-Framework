# Architecture Overview

The CRM AI Patient Simulation system is designed to emulate lifelike patient-provider conversations for training and evaluation purposes. Core components include patient agent logic (`patient_agent.py`), profile management (`patient_profiles.py`), and scenario generation (`scenario_generator.py`), each modularized to enable easy extension or refinement. The system interacts with large language models (`llm_client.py`) to drive realistic dialogue and scenario variability. Voice functionality is supported by dedicated modules (`voice_client.py`, `voice_scenario.py`), allowing conversations to be delivered and recorded in spoken form, enhancing immersion and the authenticity of simulated experiences.

Key design choices emphasize modular separation—patient logic, language model communication, and voice processing are distinct so that each part may be iterated independently. Logging and evaluation (`logs/`, `evaluation_client.py`) are centralized to facilitate thorough analysis and easy integration of analytical pipelines or quality assurance tools. The use of standard formats (e.g., JSON for logs, text for summaries) ensures interoperability and future scalability. This structure gives developers and trainers flexibility to refine components without impacting the system as a whole, safeguarding extensibility for new use cases such as additional scenario types, evaluation metrics, or voice interaction enhancements.
